# FT Engine 桌面 UI 与交互目标

> 本文是后续 UI 重构的实现规范，不代表当前页面已经完成。当前差距见 [当前架构](./ARCHITECTURE_CURRENT_zh.md)。

## 1. 主窗口

应用启动时必须居中显示为较大的普通窗口，不调用 `maximize()`、`setFullScreen(true)` 或 kiosk 模式。

建议基于当前显示器 `workArea` 计算：

- 初始宽度取工作区的 72.5%，允许范围 70%～75%。
- 初始高度取工作区的 77.5%，允许范围 75%～80%。
- 优先把宽高比约束在 1.55～1.68，目标接近 16:10。
- 常见桌面建议最小内容尺寸 `960 x 600`，但小屏启动时必须先保留至少 24px 工作区边距，不能为了最小尺寸越界。
- 设置 `resizable=true`、合理 `minWidth/minHeight`，允许用户拖动、缩放、最大化和还原。
- 每次启动默认使用计算尺寸并居中；若未来恢复上次窗口状态，只恢复未最大化且仍落在可见工作区内的边界。

主窗口保持不透明。圆角优先使用 Windows 11/macOS 系统窗口能力；柔和阴影使用 BrowserWindow 原生阴影。若继续使用 `frame:false`，必须完整实现拖动区、窗口按钮、八方向缩放、双击标题区最大化、键盘可访问性和多屏恢复。CSS 圆角不能被当作真实窗口裁剪的替代。

Overlay 是另一种窗口类型：透明、无主侧栏、无主窗口阴影约束，不能继承主窗口壳层。

## 2. 应用壳层

~~~text
┌──────────────────────────────────────────────────────────────┐
│  固定侧栏  │  顶部上下文 / 状态 / 窗口控制                  │
│            ├─────────────────────────────────────────────────┤
│  工作台    │                                                 │
│  赛事      │       受限宽度的主要工作区                      │
│  复盘      │       max-width: 1120～1280px                   │
│  设置      │                                                 │
│            │                                                 │
│ 头像 昵称  │                                                 │
└──────────────────────────────────────────────────────────────┘
~~~

- 侧栏桌面宽度建议 224～240px，固定不随页面滚动；紧凑窗口可收窄为 72px 图标栏。
- 品牌、一级导航和底部用户摘要属于侧栏；头像旁显示昵称，未登录时显示“本地用户”。
- 工作区背景铺满右侧，但实际内容用 `max-width` 和自适应内边距约束，不横向拉伸表单和列表。
- 顶部只显示当前赛事/阶段/选手、设备与保存状态、必要的页面动作和窗口控制，不放营销文案。
- 导航使用 Vue Router；刷新、返回和深链接必须恢复同一项目与筛选上下文。

## 3. 视觉系统

默认浅色主题：

| 语义 | 建议值 |
| --- | --- |
| 应用背景 | `#F4F5F3` |
| 工作区背景 | `#FAFAF8` |
| 表面 | `#FFFFFF` |
| 边框 | `#E2E4E0` |
| 主文字 | `#1D1F1C` |
| 次要文字 | `#6B706A` |
| 品牌强调 | 低饱和青绿，如 `#557A6B` |
| 危险操作 | 只在确认和错误状态使用克制红色 |

- 全产品只使用一个品牌强调色；状态色仅表达成功、警告和错误，不作为页面装饰。
- 不使用强渐变、径向光斑、毛玻璃 Hero、大面积鲜艳背景或按功能分配多种高饱和色。
- 卡片圆角不超过 8px，只用于重复实体、模态框和确需边界的工具；页面分区用留白、分隔线和全宽栏。
- 字号不随 viewport 宽度缩放。表格、工具栏和计分组件使用稳定尺寸，动态分数不引发布局跳动。
- 黑白皮肤通过语义 Token 切换；深色主题是中性黑灰，不改变信息层级或引入彩色渐变。

## 4. 导航信息架构

| 导航 | 内容 |
| --- | --- |
| 工作台 | 当前/最近赛事、继续计分、设备与存储状态 |
| 赛事 | 创建、配置、阶段、选手、裁判和设备绑定 |
| 复盘 | 历史筛选、分数、事件时间线、视频和导出 |
| 设置 | 语言、主题、快捷键、设备备注、OBS、存储和更新 |

社区只有在功能可用时显示。用户资料通过侧栏底部入口进入，不为本地功能设置登录墙。

## 5. 关键工作流

### 工作台

- 不再使用三张大入口卡片和历史模态框。
- 顶部提供“新建赛事”主操作；有活动赛事时优先显示“继续计分”。
- 最近赛事使用表格，行内提供继续、复盘、导出和更多菜单；删除放入更多菜单并二次确认。
- 设备、数据库、迁移和保存异常使用持续状态条，不以一次性 alert 代替。

### 新建与模式分流

新建流程必须先选择模式，再进入对应的最短路径。两种模式共享项目保存、裁判设备绑定和计分工作台，但不共享配置步骤。

#### 自由模式

1. 输入项目名称并选择 `FREE`。
2. 设置裁判数量；默认创建一个 `Free Mode` 组、`Player 1` 上下文和一次运行。
3. 直接进入设备扫描和裁判绑定，不显示 Stage、组别、选手名单或尝试次数。
4. 设备连接完成后进入现场计分。

自由模式的数据库会话仍保存 `stageId` 和 `attemptNumber = 1`，但这些是内部上下文，不在前端作为配置项展示。后续切换自由模式选手时只增加轻量选手上下文，不重新引入赛事向导。

#### 赛事模式

1. 输入项目名称并选择 `TOURNAMENT`。
2. 配置一个或多个 Stage、组别、选手和裁判。
3. 为 Stage 配置 `1..20` 次尝试，并在运行前选择 Stage、组别、选手和尝试。
4. 在同一上下文完成设备扫描、绑定和连接检查。
5. 进入现场计分。

### 赛事配置

- 步骤导航固定可见，每步自动保存并显示保存状态。
- 设备绑定在裁判行内完成，扫描、重命名、连接状态和冲突原因保持同一上下文。
- 开始比赛前提供检查清单；错误项可点击跳回对应步骤。
- “返回”和“取消赛事”语义分开，避免退出配置时意外丢失草稿。
- 自由模式只保留项目名称和裁判数量；赛事模式才显示 Stage、组别、选手和尝试次数字段。
- 尝试次数在自由模式固定为 1；赛事模式只有在存在多次尝试时才在计分和复盘上下文显示。

### 现场计分

- 始终显示赛事、阶段、组别、当前选手、设备状态和数据库保存状态。
- 主要操作固定为暂停/继续、归零、上一位、完成并下一位和结束；危险动作不与高频动作相邻。
- 切换选手必须先提交当前 MatchSession，再原子切换上下文和 Reset；处理中禁用重复点击。
- 设备断开不自动切换选手；保留最后持久化分数并提供明确重连动作。
- 不常驻显示视频链接输入框和大播放器。由悬浮/工具菜单中的“绑定或打开视频”进入 URL 输入，再打开可停靠播放器。
- 播放器旁可显示与 OBS 悬浮模式一致的紧凑计分组件；播放器关闭不清除媒体绑定或事件。

### 复盘

- 项目、阶段、组别、选手筛选保持在页面顶部或左侧筛选栏，不用层层模态框；自由模式隐藏无意义的 Stage/attempt 筛选，只显示项目、选手和裁判上下文。
- 视频、事件时间线和裁判筛选共享上下文；点击事件切换视频并 seek 后，列表焦点和筛选不丢失。
- 视频载入后显示同一套悬浮计分组件，并随播放时间重建分数。
- 视频不可嵌入、删除或断网时保留事件列表，提供“在浏览器中打开”，不让播放器错误覆盖整页。
- “未对齐”事件不可点击成伪跳转，要说明 stale、context mismatch、not ready 等原因。

### 赛事与复盘视觉迁移

- 赛事列表、赛事配置、设备绑定、现场计分和复盘统一使用 Workbench 背景、表面、边框、主次文字、品牌强调和业务状态色。
- 清理旧黑色页面中的独立背景、按钮颜色、输入框边框和局部阴影；不改变 Overlay 的透明窗口和独立布局。
- 页面分区使用留白和分隔线，卡片只用于重复实体、设备绑定项、事件项和 Dialog；不把页面整体套成卡片。
- 设备扫描失败显示稳定错误码对应的本地化提示、重试动作和必要的服务重启动作；不把 `An object could not be cloned.` 作为用户界面文案。

## 6. 设置和反馈

- 当前全宽下拉设置改为独立设置页；仅主题等快速项可放小型菜单。
- 用应用内 Dialog 替代原生 `alert`/`confirm`，支持焦点锁定、Esc、Enter 和清晰的主次按钮。
- 短暂成功使用 toast；持续错误使用状态条或行内错误；需要用户决策才使用 Dialog。
- 删除数据必须显示影响范围，要求再次确认，并在完成后展示未能删除的具体路径。
- 自动更新下载完成后进入通知中心或设置页，不在页面加载后突然弹出系统确认框。

## 7. 响应式和验收

至少验证 1366x768、1920x1080、2560x1440 和双显示器不同缩放。每个尺寸检查：

- 启动窗口居中、未最大化、未全屏，四周有明显桌面边距。
- 侧栏、窗口控制和拖动区域可用，最大化/还原后布局不重叠。
- 主工作区不过宽，最长中英文/日文文案不溢出按钮或表格。
- 计分、视频、悬浮计分和事件列表在最小尺寸下仍可操作。
- 自由模式从项目配置到设备绑定不超过两个用户步骤，且不存在 Stage、组别、选手名单和尝试次数控件。
- 赛事模式可以恢复 Stage、组别、选手和尝试上下文；刷新、返回和深链接不会丢失上下文。
- 设备扫描异常只显示稳定本地化错误和可执行重试，不出现未处理的 DataCloneError 或对象序列化文本。
- 浅色和深色主题均满足对比度，焦点环清晰。
- Overlay 透明区域、点击穿透和 OBS 捕获不受主壳层改动影响。
