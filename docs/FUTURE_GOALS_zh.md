# FT Engine 后续目标

本文记录尚未纳入稳定版本的产品与工程目标，不追踪历史重构进度，也不约束具体实现目录。

## 产品目标

- 完善自由模式与赛事模式的创建、恢复、计分、复盘和导出流程。
- 在 Windows 和 macOS 上提供一致的 BLE、USB、Overlay、媒体播放和快捷键体验。
- 增加可选的账号与社区服务；网络服务不可用时，本地赛事仍可完整运行。
- 为 OBS、报表和复盘提供稳定的导出格式与版本兼容策略。

## 工程目标

- 继续保持 Renderer 只通过受控 IPC 访问本地能力，领域逻辑与平台实现解耦。
- 将平台权限、设备重连、睡眠恢复、窗口跟踪和安装签名纳入持续集成与实机验收。
- 为 SQLite 崩溃恢复、设备协议和导出文件建立长期回归测试。
- 在引入远程账号服务时，明确本地数据与云端用户数据的边界，不上传本地赛事明细。

## 验收重点

发布前应覆盖：全新数据库初始化与恢复、BLE/USB 扫描和重连、多显示器 Overlay、YouTube 复盘、CSV/SRT/ZIP 导出、权限拒绝和应用退出清理。

具体平台约束见 [Windows 与 macOS 平台适配规范](./PLATFORM_ADAPTATION_zh.md)，服务端边界见 [Django 用户服务目标](./BACKEND_DJANGO_zh.md)。
