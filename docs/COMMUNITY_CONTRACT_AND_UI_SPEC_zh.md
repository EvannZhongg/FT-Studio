# FT Engine 用户接口与桌面产品规范

> 本文描述目标产品形态，不代表当前实现进度。当前应用仍使用旧首页、单一 Pinia Store 和部分 localhost API；实际迁移状态与阻断项以 [路线 B 重构状态](./REFACTOR_PLAN_ROUTE_B_zh.md) 为准。

## 1. 目的

本文档定义独立用户后端上线前的临时登录方案，以及重构后桌面端的核心页面和赛事流程。

范围包括：

- 临时测试用户和未来用户后端边界。
- 工作台、赛事阶段、现场计分、赛后复盘和用户资料。
- 社区功能的预留方式。

本文档只描述平台无关逻辑。系统权限、BLE/USB、OBS、打包和签名差异见 [Windows 与 macOS 平台适配规范](./PLATFORM_ADAPTATION_zh.md)。

## 2. 产品原则

- 本地赛事、计分、复盘和导出不要求登录。
- 登录只用于用户资料、社区浏览和发布。
- 云端不可用不影响本地功能。
- 启动后直接进入工作台，不设置登录墙或营销首页。
- 社区未实现时通过功能开关隐藏，不展示不可用占位页。

## 3. 后端未上线期间的接口策略

桌面端使用可替换 Gateway：

~~~text
Renderer
   │ typed IPC
Electron Main AuthService
   ├─ DevMockCommunityGateway
   └─ HttpCommunityGateway -> Independent Server
~~~

Renderer 只依赖账号 IPC，不感知 Mock 或真实服务器。真实后端上线时替换 Gateway，不修改登录页和资料页。

### 3.1 账号能力

桌面端先固定以下能力：

| 能力 | 说明 |
| --- | --- |
| 获取会话 | 返回未登录、Mock 登录或真实登录状态 |
| 登录 | 输入邮箱和密码，由 Electron Main 处理 |
| 退出 | 清理当前会话和安全存储 |
| 获取资料 | 返回用户 ID、邮箱、显示名、头像、简介和语言 |
| 更新资料 | 第一阶段支持显示名、简介和语言 |
| 会话变化事件 | 登录、退出或会话失效时通知 Renderer |

Access Token 和 Refresh Token 不得通过 IPC 返回 Renderer。所有 IPC 输入输出使用共享 Schema 校验。

### 3.2 临时测试用户

仅开发环境提供：

~~~text
Email:    demo@ft-engine.local
Password: demo-only
Name:     FT Engine Demo
~~~

限制：

- 只有开发模式且显式启用 Mock Auth 时可登录。
- 打包版本必须拒绝启用 Mock Gateway。
- 登录页的“填入测试账号”只在开发环境出现。
- Mock 用户可测试登录、退出和资料编辑。
- Mock 用户不能真实发布，发布时返回“社区服务未启用”。
- Mock 密码、Token 和用户操作不得写入日志。

## 4. 建议的独立用户后端

第一期使用 NestJS 模块化单体，先实现认证、用户、资料、会话和健康检查。帖子模块后续加入同一服务，不提前拆微服务。

建议首批接口：

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| GET | /v1/health | 服务健康状态 |
| POST | /v1/auth/login | 登录 |
| POST | /v1/auth/refresh | 刷新并轮换会话 |
| POST | /v1/auth/logout | 退出当前会话 |
| GET | /v1/me | 当前用户资料 |
| PATCH | /v1/me/profile | 更新资料 |
| GET | /v1/me/sessions | 登录设备 |
| DELETE | /v1/me/sessions/:id | 撤销设备会话 |

后端建议使用 PostgreSQL，至少区分用户、身份、资料和会话数据。密码只保存 Argon2id 哈希；Refresh Token 轮换并支持撤销。

错误响应使用稳定错误码，例如：

- AUTH_INVALID_CREDENTIALS
- AUTH_SESSION_EXPIRED
- AUTH_ACCOUNT_SUSPENDED
- AUTH_RATE_LIMITED
- PROFILE_INVALID
- COMMUNITY_NOT_AVAILABLE
- NETWORK_UNAVAILABLE

Renderer 根据错误码显示本地化文案，不直接展示服务器原始错误。

## 5. 主导航和应用壳

采用固定侧边栏和顶部状态区：

| 导航 | 作用 | 目标首期范围 |
| --- | --- | --- |
| 工作台 | 当前赛事、最近赛事、设备状态 | 目标能力 |
| 赛事 | 创建、配置和执行赛事 | 目标能力 |
| 复盘 | 查看分数、事件、视频和导出 | 目标能力 |
| 社区 | Feed、发布和互动 | 功能开关隐藏 |
| 用户资料 | 登录、资料、本地统计和会话 | 目标能力 |
| 设置 | 语言、设备、快捷键、存储和更新 | 目标能力 |

用户头像和登录状态放在侧边栏底部。BLE/USB 总体状态放在顶部状态区，避免设备异常只在计分页可见。

界面保持工作型布局：

- 使用表格、分栏和固定工具栏。
- 不使用大面积 Hero 或装饰性卡片。
- 动态分数不能导致布局跳动。
- OBS、设备断开和保存失败使用持续状态提示。

## 6. 工作台

工作台是默认页面，账号恢复在后台进行，不阻塞本地内容。

主要内容：

1. 有进行中赛事时显示“继续计分”、赛事上下文和 OBS 快捷操作。
2. 没有进行中赛事时提供“新建赛事”。
3. 最近赛事使用表格，支持继续配置、进入复盘和打开数据位置。
4. 显示设备已发现、已连接和异常数量。
5. 登录状态只占次要位置。

工作台不承担详细编辑。赛事配置进入赛事模块，数据分析进入复盘模块。

## 7. 赛事和计分阶段

### 7.1 数据层级

~~~text
赛事 Competition
└─ 计分阶段 Stage
   └─ 组别 Group
      └─ 选手 Contestant
         └─ 计分会话 MatchSession
            └─ 计分事件 ScoreEvent
~~~

Stage 是正式领域对象，可表示资格赛、半决赛、决赛或自定义阶段，并支持排序。

第一期阶段能力：

- 添加、删除、重命名和排序。
- 分别配置组别、选手、尝试次数和裁判。
- 手动从上一阶段复制或选择晋级选手。
- 同一赛事同时只有一个进行中阶段。

第一期不实现复杂自动晋级规则，避免扩大重构范围。

### 7.2 兼容现有数据

- 自由模式迁移为一个“自由计分”阶段。
- 赛事模式迁移为一个“主赛事”阶段。
- 原有 Group、选手、裁判和顺序保持不变。
- 迁移过程记录版本并可重试。

### 7.3 状态

| 对象 | 状态 |
| --- | --- |
| 赛事 | 草稿、就绪、进行中、已完成、已归档 |
| 阶段 | 草稿、就绪、进行中、已完成 |
| 计分会话 | 待开始、进行中、暂停、已完成、已作废 |

已完成会话不直接修改原始事件。需要纠正时作废并重新计分，保留历史记录。

## 8. 赛事配置流程

新建赛事使用自动保存的步骤流程：

1. 基本信息：名称、日期、地点和模板。
2. 计分阶段：添加并排序资格赛、半决赛、决赛或自定义阶段。
3. 组别与选手：配置选手顺序和尝试次数。
4. 裁判与设备：配置裁判、SINGLE/DUAL 和 BLE/USB 绑定。
5. 显示与媒体：配置 OBS 和可选 YouTube。
6. 赛前检查：验证阶段、选手、设备、裁判和存储状态。

检查通过后赛事才能进入就绪状态。未完成配置可以保存为草稿。

## 9. 现场计分

页面固定显示：

- 顶部：赛事、阶段、组别、当前选手和总体设备状态。
- 左侧：待计分、进行中、已完成和作废的选手队列。
- 中间：多裁判实时分数和波形。
- 右侧：设备、媒体同步和 OBS 状态。
- 底部：暂停、归零、上一位、完成并下一位、结束阶段。

关键交互：

- “完成并下一位”先提交当前会话，再切换选手和 Reset。
- 结束阶段前检查未完成选手并要求确认。
- 设备断开时保留已保存分数，不自动切换选手。
- 修改进行中阶段的规则、裁判或选手需要暂停并确认。
- OBS 可随时打开、定位、切换点击穿透或关闭。
- YouTube 是辅助视图，不替代核心计分界面。

## 10. 赛后复盘和导出

复盘按赛事、阶段、组别、选手和裁判筛选，包含：

| 视图 | 内容 |
| --- | --- |
| 总览 | 阶段状态、完成率、会话数和异常记录 |
| 分数 | 折算分、原始分、裁判明细和规则版本 |
| 事件时间线 | 加分、减分、扣分、Reset 和设备来源 |
| 视频回放 | 根据媒体时间重建当时分数 |
| 导出 | 选择范围和输出格式 |

导出由 Electron Main 调用系统保存对话框完成，第一期保留：

- 汇总 CSV/XLSX。
- 原始事件 CSV。
- TXT、SRT 和组合 ZIP。

导出范围支持整个赛事、单一阶段、组别或选手。失败时保留选项，并显示路径权限、磁盘空间等可操作原因。

## 11. 用户资料

### 11.1 未登录

- 显示登录表单。
- 明确本地功能无需登录。
- 开发环境可填入测试账号，正式版本隐藏。

### 11.2 已登录

- 编辑头像、显示名、简介和语言。
- 查看本地赛事与计分统计。
- 查看登录方式和设备会话。
- 提供退出当前设备和退出所有设备。
- 社区上线后再显示帖子和互动统计。

邮箱作为身份字段，第一期不在普通资料表单中修改。头像上传在对象存储上线后启用。

### 11.3 离线

- 显示最后缓存的非敏感资料和离线状态。
- 允许查看本地统计，不提交云端资料修改。
- Token 无法刷新时不阻止进入工作台、赛事和复盘。

## 12. 社区预留

帖子第一阶段不实现，只预留：

- /community 路由和功能开关。
- 帖子摘要、详情和创建输入的契约目录。
- 本地分享草稿和上传任务。
- 从复盘创建分享草稿的服务入口。

功能关闭时：

- 隐藏社区导航和“发布到社区”按钮。
- 保留本地导出。
- 不在正式版本展示空 Feed 或“敬请期待”页面。

未来社区可包含发现、关注、我的发布和草稿上传。帖子只允许受限 Markdown、规范化视频链接、图片和计分快照，不接受任意 HTML。

## 13. 核心流程

本地赛事：

~~~text
工作台 -> 新建赛事 -> 配置阶段和设备 -> 赛前检查
       -> 现场计分 -> 完成阶段/赛事 -> 复盘 -> 导出
~~~

未来发布：

~~~text
复盘 -> 创建分享草稿 -> 选择范围和隐私
     -> 登录检查 -> 上传 -> 发布完成
~~~

上传失败只影响上传任务，不改变本地赛事状态。

## 14. 页面权限

| 功能 | 未登录 | Mock 用户 | 真实用户 |
| --- | --- | --- | --- |
| 工作台、赛事、计分 | 允许 | 允许 | 允许 |
| 复盘和导出 | 允许 | 允许 | 允许 |
| 查看本地统计 | 允许 | 允许 | 允许 |
| 修改账号资料 | 需登录 | 本地 Mock | 允许 |
| 发布分享 | 需登录 | 不可发布 | 允许 |

登录守卫只用于云端操作，不对整个应用设置全局登录墙。

## 15. 实施顺序

本文不维护工程阶段清单。桌面端的当前进度、下一步顺序和切换门槛统一见 [路线 B 重构状态](./REFACTOR_PLAN_ROUTE_B_zh.md)，避免产品规范与代码进度重复并产生冲突。

账号和社区仍排在本地主链路完成之后：先完成 Electron Main、SQLite、Platform Worker 和 Renderer IPC 切换，再接入账号 Gateway，最后启用社区发布与互动。

## 16. 第一阶段验收

- 用户服务器未上线时，开发版可使用 Mock 用户测试登录和资料。
- 正式构建无法启用 Mock 登录。
- 未登录用户可完成多阶段赛事、计分、复盘和导出。
- 当前赛事、阶段、组别和选手上下文始终清晰。
- 工作台可继续进行中赛事并打开历史复盘。
- 社区关闭时不出现不可用导航或发布按钮。
- Renderer 不持有 Token，也不直接调用用户后端。
- OBS、BLE、USB、YouTube 和现有导出能力不因账号层退化。
- 真实后端上线时只替换 Gateway，不重写 Renderer 页面。
