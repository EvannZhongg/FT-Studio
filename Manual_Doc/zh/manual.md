# FT Engine 使用说明书

## 1. 软件简介

FT Engine 是一套用于电子裁判计分的桌面软件，核心能力包括：

1. 新建比赛并配置自由模式或赛事模式。
2. 扫描并绑定蓝牙计分设备。
3. 在比赛过程中实时显示每位裁判的分数状态。
4. 使用悬浮窗在全屏或指定窗口上叠加显示比分。
5. 为选手绑定 YouTube 视频，在视频旁显示实时或历史悬浮分数。
6. 保存比赛过程数据，并在赛后查看报表与导出明细文件。

![](\images\01.png)

---

## 2. 使用前准备

开始使用前，建议先确认以下事项：

1. 电脑蓝牙已开启，且可正常扫描到裁判设备。
2. 裁判设备处于可连接状态。
3. 如需使用悬浮窗，请提前打开目标应用或比赛画面窗口。
4. 如需导入选手名单，请准备好 `.csv`、`.xlsx` 或 `.xls` 文件。
5. 如需直播或录屏，请先确认是否需要开启“OBS 隔离”。
6. 如需使用 YouTube 视频计分或视频复盘，请确认网络可访问 YouTube，且视频允许嵌入。


### 2.1 软件内的主要页面

| 页面 | 作用 |
| --- | --- |
| 首页 | 新建比赛、查看历史记录、进入视频复盘、继续或删除项目 |
| 顶部设置面板 | 语言切换、快捷键设置、OBS 隔离、本地数据清理 |
| 赛前设置向导 | 创建项目、配置组别、导入名单、绑定设备 |
| 计分页 | 实时计分、切换选手、归零、开启悬浮窗、结束比赛 |
| 悬浮窗 | 在全屏或指定窗口上显示比分与波形图 |
| 视频计分 | 播放当前选手视频，并在旁边显示实时悬浮分数 |
| 视频复盘 | 按项目、组别、选手和裁判查看视频计分事件 |
| 报表页 | 查看折算分、原始分、导出 CSV / ZIP / SRT 明细 |

## 3. 顶部栏与全局设置

软件顶部栏在非悬浮窗模式下始终可见，包含品牌区、设置按钮和窗口控制按钮。


### 3.1 顶部栏可执行的操作

1. 单击右上角设置按钮，展开全局设置面板。
2. 单击最小化按钮，将主窗口缩小到任务栏。
3. 单击最大化按钮，切换窗口最大化或还原。
4. 单击关闭按钮，关闭主窗口；如果悬浮窗已打开，也会一并关闭。
5. 双击顶部栏空白区域，也可以最大化或还原窗口。

![](\images\02.png)

### 3.2 设置面板可执行的操作

#### 3.2.1 语言切换

1. 打开设置面板。
2. 在“语言”下拉框中选择 `简体中文`、`English` 或 `日本語`。
3. 语言会立即切换，并保存到本地设置。

![](\images\03.png)

#### 3.2.2 重置快捷键设置

1. 打开设置面板。
2. 单击“重置快捷键”输入框。
3. 直接按下你希望使用的组合键，例如 `Ctrl+G`、`Ctrl+1`。
4. 软件会立刻保存这个快捷键。

说明：

1. 该快捷键是全局快捷键，在计分页中会被注册。
2. 当计分页处于“自动”状态时，快捷键执行“下一位”逻辑。
3. 当计分页关闭“自动”时，快捷键执行“归零”逻辑。

![](\images\04.png)

#### 3.2.3 OBS 隔离

1. 打开设置面板。
2. 在“OBS 隔离”中选择“启用”或“不启用”。

作用说明：

1. 启用后，主窗口内容不会被 OBS 捕获。
2. 悬浮窗不受此设置影响，仍可用于直播叠加显示。

适用场景：

1. 只希望直播中展示悬浮比分，不希望观众看到主控界面。

![](\images\05.png)

#### 3.2.4 删除所有本地数据

1. 打开设置面板。
2. 单击“删除所有本地数据”按钮。
3. 在确认弹窗中再次确认。
4. 软件会删除本机保存的比赛记录、应用设置和日志，并自动重启。

注意事项：

1. 该操作不可撤销。
2. 被删除的数据包括本地设置、legacy `match_data`、SQLite 数据库与备份、导出目录和日志。
3. 如果删除失败，通常需要先关闭程序后重试。

![](\images\06.png)

## 4. 首页

首页是进入软件后的默认页面，主要用于开始新比赛和管理历史项目。


### 4.1 首页可执行的操作

1. 单击“新建比赛”，进入赛前设置向导。
2. 单击“历史记录”，打开历史项目列表。
3. 单击“视频复盘”，进入独立的视频事件复盘页面。

![](\images\07.png)

### 4.2 新建比赛

1. 单击“新建比赛”卡片。
2. 软件会先清空当前前端缓存状态。
3. 然后进入赛前设置向导的第 1 步。

适合场景：

1. 开始一场全新的比赛。
2. 不希望沿用旧项目中的组别、名单和设备绑定配置。


### 4.3 历史记录列表

单击“历史记录”后，会弹出历史项目列表窗口。


每条历史记录都可以执行以下操作：

1. `查看`
   作用：直接打开该项目的报表页。
2. `继续`
   作用：加载该项目配置，并回到赛前设置向导，便于继续比赛或重新绑定设备。
3. `删除`
   作用：删除该项目的全部历史数据。

![](\images\08.png)

注意事项：

1. “继续”会恢复项目名称、模式、组别、选手名单和已保存的裁判绑定配置。
2. “删除项目”是不可撤销操作。
3. 从报表页返回首页后，首页会默认重新打开历史记录弹窗，方便继续查看其他项目。

## 5. 赛前设置向导

赛前设置向导分为 2 到 3 个步骤，具体取决于你选择的是自由模式还是赛事模式。


### 5.1 第一步：基础设置

这一页用于创建项目和确定比赛模式。

![](\images\09.png)

#### 5.1.1 可执行的操作

1. 在“项目名称”中输入比赛名称。
2. 在“比赛模式”中选择：
   `自由模式 (Free Mode)` 或 `赛事模式 (Tournament)`。
3. 如果选择自由模式，需要填写“裁判人数”。
4. 单击“下一步”进入后续流程。
5. 单击“取消”返回首页。


#### 5.1.2 两种模式的区别

**自由模式**

1. 不需要先配置多个组别。
2. 系统会自动创建一个默认组别 `Free Mode`。
3. 适合临时测试、快速打分、临时追加选手。
4. 在比赛过程中，当所有已有选手完成后，可以继续新增选手。

**赛事模式**

1. 进入下一步“组别管理”。
2. 适合正式比赛，有明确组别、名单和裁判数量。
3. 全部选手完成后，可以选择从第一位重新开始。


#### 5.1.3 使用注意事项

1. 项目名称会用于生成项目目录，建议填写明确的比赛名。
2. 自由模式下的裁判人数会直接决定后续设备绑定卡片数量。
3. 如果是从“历史记录 -> 继续”进入，这一页会自动回填已有项目配置。

---

### 5.2 第二步：组别管理

这一页只会在“赛事模式”下出现，用于配置比赛组别、裁判数量和选手名单。

![](\images\10.png)

#### 5.2.1 左侧组别列表可执行的操作

1. 查看当前已创建的组别。
2. 单击某个组别，切换右侧编辑区。
3. 单击“新增组别”，创建新的组别。

![](\images\11.png)

#### 5.2.2 右侧配置详情可执行的操作

1. 修改组别名称。
2. 设置该组的裁判数量。
3. 在文本框中输入选手名单，每行一位。
4. 单击“导入名单”，从 Excel 或 CSV 文件中导入名单。
5. 当组别数大于 1 时，可以删除当前组别。

![](\images\12.png)

#### 5.2.3 导入名单的操作流程

1. 单击“导入名单”按钮。
2. 选择 `.csv`、`.xlsx` 或 `.xls` 文件。
3. 软件会读取第一个工作表，并识别所有非空列。
4. 如果检测到多列候选数据，会弹出“选择要导入的列”窗口。
5. 选择包含选手姓名的一列。
6. 单击“确认导入”。
7. 导入内容会追加到当前组别的选手名单文本框中。

![](\images\13.png)

#### 5.2.4 保存并进入下一步

1. 单击“保存并下一步”。
2. 软件会把当前每个组别的名单按换行切分为正式选手列表。
3. 然后进入设备绑定页。


#### 5.2.5 使用注意事项

1. 文本框中的空行会被自动忽略。
2. 组别名称建议填写明确，后续报表页会直接使用。
3. 导入名单时，软件只会从你选中的一列读取非空内容。
4. 如果当前项目是从历史记录继续打开，已有组别和名单会自动回填。

---

### 5.3 第三步：设备绑定

这一页用于扫描设备、设置裁判名称、选择单机或双机模式，并将设备与裁判卡位绑定。

![](\images\14.png)

#### 5.3.1 页面顶部可执行的操作

1. 查看当前扫描状态。
2. 查看已发现设备数量。
3. 单击“设备备注设置”按钮，为设备添加备注名。
4. 单击“刷新设备”，重新扫描蓝牙设备。
5. 在赛事模式下，可以切换当前要启动的组别。

![](\images\15.png)

#### 5.3.2 每个裁判卡片可执行的操作

1. 修改裁判显示名称。
2. 选择模式：
   `单机` 或 `双机`。
3. 选择主设备。
4. 如果是双机模式，再选择副设备。

![](\images\16.png)

说明：

1. 已经被其他裁判占用的设备，不会再次出现在下拉列表中。
2. 切回“单机”模式时，副设备会被自动清空。
3. 单机模式下，软件显示该设备的 `+`、`-` 和总分。
4. 双机模式下，软件同时接收主设备和副设备数据。
5. 双机模式会额外显示“重点扣分”。
6. 成绩单中的重点扣分计算只对双机裁判生效。


#### 5.3.4 设备备注设置

单击顶部标签按钮后，会打开“设备备注设置”弹窗。

可执行的操作：

1. 查看已扫描到的设备原始名称和地址。
2. 为每个设备填写自定义备注。
3. 单击“保存”。

保存后的效果：

1. 备注会保存到全局设置中。
2. 后续设备下拉框会优先显示为“备注名 (设备原名)”。
3. 备注是全局生效的，不只针对当前项目。

![](\images\17.png)

#### 5.3.5 永久修改设备蓝牙名称

当某些设备备注发生变化后，软件会弹出“永久修改设备名称”确认窗口。

你可以执行的操作：

1. 勾选要写入设备蓝牙名称的项目。
2. 单击“写入所选设备”，把备注名永久写入硬件蓝牙名称。
3. 单击“暂不修改”，只保留本地备注，不改硬件名称。

注意事项：

1. 即使永久重命名失败，本地备注仍然会保留。
2. 只有当前能被识别到的设备，才可以执行永久重命名。


#### 5.3.6 开始比赛

1. 确认每位裁判的名称、模式、主设备、副设备都已设置正确。
2. 单击“开始比赛”。
3. 软件会保存当前组别配置，并进入设备连接状态。


---

### 5.4 设备连接状态弹窗

单击“开始比赛”后，会出现“正在连接设备...”弹窗。


你可以在这里看到：

1. 每位裁判的名称。
2. 主设备连接状态标签。
3. 双机模式下的副设备连接状态标签。

状态可能包括：

1. `waiting`
2. `connecting`
3. `connected`
4. `error`

#### 5.4.1 弹窗中的操作

1. 如果设备正常连接，软件会自动进入计分页。
2. 如果 8 秒后仍未全部连接成功，会显示“连接超时”提示。
3. 此时可以单击“取消”，终止本次比赛启动。
4. 也可以单击“强制进入”，直接进入计分页。

使用建议：

1. 正式比赛前，建议尽量确认所有设备都变为 `connected` 再进入。
2. “强制进入”适合调试、测试或临时跳过某些设备时使用。

## 6. 计分页

计分页是比赛过程中的主控页面，用于观察实时分数、切换选手、归零、启动悬浮窗和结束比赛。

![](\images\18.png)

### 6.1 页面顶部左侧：结束比赛

1. 单击“结束”按钮。
2. 软件会向后端发送停止比赛命令。
3. 所有裁判连接会被断开。
4. 页面会返回首页。


注意事项：

1. 已经写入的历史数据不会因为“结束”而丢失。
2. 结束后如果需要继续同一个项目，可从首页历史记录中重新进入。

### 6.2 页面顶部中间：当前组选手切换

中间区域显示当前组别和当前选手。

你可以执行的操作：

1. 查看当前组别名称。
2. 使用左箭头切换到上一位选手。
3. 使用右箭头切换到下一位选手。
4. 使用下拉框直接选择任意选手。
5. 已经完成打分的选手会在下拉列表中显示勾号。

![](\images\19.png)

注意事项：

1. 手动切换选手时，软件会同步更新当前比赛上下文。
2. 切换选手后，软件会执行一次分数归零。
3. 自由模式下，切到最后一位之后继续“下一位”，系统会自动新增 `Player N`。

### 6.3 页面顶部右侧：自动、悬浮、下一位、归零

#### 6.3.1 自动开关

1. 单击“自动”按钮，可切换启用或关闭。

启用时：

1. “下一位”会直接保存当前选手得分并切换到下一位。
2. 全局快捷键也会执行同样逻辑。

关闭时：

1. 单击“下一位”会先弹出确认框。
2. 全局快捷键改为执行“归零”。


#### 6.3.2 选择悬浮方式

1. 单击“悬浮”按钮。
2. 在弹窗中选择“窗口悬浮”或“YouTube 视频计分”。

选择“窗口悬浮”时：

1. 选择目标窗口或“全屏模式”。
2. 单击“启动悬浮窗”。
3. 软件会打开一个独立的透明悬浮窗。

![](\images\20.png)

注意事项：

1. 软件会读取当前系统窗口标题列表。
2. 如果选择指定窗口，悬浮窗会按照该窗口当前的坐标和尺寸打开。
3. 当前代码中悬浮窗打开后不会持续自动跟随目标窗口移动，若目标窗口位置变化，需要手动拖动悬浮窗调整。

#### 6.3.3 YouTube 视频计分

1. 在悬浮方式弹窗中选择“YouTube 视频计分”。
2. 输入当前选手的公开或不公开且允许嵌入的 YouTube 链接。
3. 单击“进入播放器”。
4. 页面进入“视频 + 悬浮计分”布局；播放器时间会与计分事件同步。
5. 使用右上角链接图标可以更换当前选手的视频，使用返回图标退出播放器布局。

视频绑定按“组别 + 选手”保存。切换到另一位选手时会加载其独立绑定；未绑定时可从播放器区域重新进入绑定弹窗。

播放器旁的悬浮计分支持“仅总分”“分项”和“组合显示”。该显示只改变界面，不改变原始计分数据。

#### 6.3.4 下一位

1. 单击“下一位”按钮。
2. 当前选手会被标记为“已完成打分”。
3. 软件会寻找当前组中下一位尚未完成的选手。
4. 找到后切换过去，并自动归零。

当所有选手都完成后：

1. 自由模式下，会继续新增下一位选手。
2. 赛事模式下，会弹出“全部完成”窗口。


#### 6.3.5 归零

1. 单击“归零”按钮。
2. 当前所有裁判的分数会重置为 0。

如果未开启“跳过确认”：

1. 会出现确认弹窗。
2. 可以勾选“不再询问”。


### 6.4 裁判分数卡片

页面中间的每张分数卡代表一位裁判。

卡片中可以查看：

1. 裁判名称。
2. 主设备连接状态点。
3. 双机模式下的副设备连接状态点。
4. 当前总分。
5. 当前 `+` 分与 `-` 分。
6. 双机模式下的“重点扣分”。

![](\images\21.png)

状态点颜色说明：

1. 绿色：已连接。
2. 其他颜色：等待中、连接中或异常。

### 6.5 全部完成弹窗

当当前组所有选手都完成后，会弹出“全部完成”弹窗。

![](\images\22.png)

在这里可以执行：

1. `保存并退出`
   作用：结束比赛并返回首页。
2. `继续`
   自由模式下：继续并新增选手。
   赛事模式下：从第一位选手开始重新打分。

注意事项：

1. 赛事模式中的“继续”适合重打整组。
2. 自由模式中的“继续”更适合现场临时追加选手。

## 7. 悬浮窗

悬浮窗是一个独立窗口，用于在比赛画面上叠加显示比分内容，适合 OBS、投屏或现场大屏。

![](\images\23.png)

### 7.1 悬浮窗的基本特点

1. 默认始终置顶。
2. 默认不显示任务栏图标。
3. 默认允许鼠标穿透，避免遮挡目标画面操作。
4. 当鼠标移到顶部停靠区或卡片上时，会自动进入可交互状态。

### 7.2 顶部停靠条可执行的操作

将鼠标移动到悬浮窗顶部，会出现控制停靠条。

![](\images\24.png)

可执行的操作如下：

1. 查看当前选手名称。
2. 单击 `Next`，切换到下一位选手。
3. 单击 `R`，重置当前分数。
4. 单击波形按钮，显示或隐藏波形图。
5. 单击设置按钮，打开悬浮窗设置面板。
6. 单击关闭按钮，关闭悬浮窗。

注意事项：

1. 点击 `Next` 时，当前选手同样会被标记为已完成。
2. 停靠条本身可以左右拖动，但 Y 轴固定在顶部。

### 7.3 裁判卡片可执行的操作

每个裁判卡片都可以独立拖动位置。


你还可以：

1. 拖动右下角缩放手柄，调整裁判卡片大小。
2. 调整任意一张裁判卡片尺寸时，其他裁判卡片会同步为相同尺寸。
3. 在分数字体区域按下鼠标后滚动滚轮，统一放大或缩小所有裁判卡片中的字体。


### 7.4 波形图

悬浮窗支持显示波形图部件。

波形图特点：

1. 会在当前选手开始出现非零分数后开始记录。
2. 横轴为时间，纵轴为分数。
3. 每位裁判对应一条折线。
4. 切换选手后，波形图会自动重置。
5. 所有裁判分数被清零时，波形图也会重置。
6. 波形图卡片本身可拖动、可自由缩放。

### 7.5 悬浮窗设置面板

单击停靠条中的设置按钮后，可以打开设置面板。

可设置内容如下：

1. `显示模式`
   可选：
   `分项 (+ / -)`、`仅总分`、`组合显示`、`实时 (连击)`。
2. `不透明度`
   用滑块调整背景透明程度。
3. `背景色`
   自定义卡片背景颜色。
4. `恢复默认`
   将悬浮窗样式恢复到默认配置。
5. `关闭`
   收起设置面板。


四种显示模式说明：

1. `分项 (+ / -)`：显示正分和负分的分项结构。
2. `仅总分`：只显示总分，更适合大屏展示。
3. `组合显示`：同时显示总分与简化分项。
4. `实时 (连击)`：显示短时间内累计的分数变化，更适合做动态字幕或即时反馈。

注意事项：

1. 悬浮窗样式设置会保存在本地，下次打开时会沿用。
2. 当前实现中，字体缩放是全局共享的，不区分单个裁判卡。

## 8. 报表页

报表页用于查看某个历史项目的赛果、计算折算分、查看原始分，并导出 CSV、日志和字幕文件。

![](\images\25.png)

### 8.1 左侧分组栏

左侧显示当前项目下的所有组别。

可执行的操作：

1. 单击任意组别，切换当前报表内容。
2. 单击“返回列表”，返回首页历史记录状态。


### 8.2 顶部工具栏

顶部工具栏根据当前视图提供不同操作。


可执行的操作：

1. `高级设置`
   仅在“折算分”视图下显示。
2. `导出详情`
   导出选手日志和字幕文件 ZIP。
3. `表格 CSV`
   导出当前表格视图的 CSV 文件。
4. `折算分 / 原始分`
   切换报表查看模式。

### 8.3 折算分视图

折算分视图会显示：

1. 排名。
2. 选手名称。
3. 每位裁判的折算分。
4. 重点扣分列。
5. 最终得分。

#### 8.3.1 折算规则

根据当前代码，折算逻辑如下：

1. 先找出每位裁判在当前组中的最高原始分。
2. 每位选手该裁判的分数按以下方式折算：
   `该选手原始分 / 该裁判最高分 × 比例值`
3. 默认比例值为 `60`。
4. 最终得分为所有裁判折算分的平均值。
5. 如果启用了“显示重点扣分”，则最终得分还会减去标准重点扣分。

#### 8.3.2 重点扣分规则

根据当前实现：

1. 只统计双机模式裁判的重点扣分。
2. 采用“多数优先”原则。
3. 如果多个扣分值出现次数相同，则取较大值。

### 8.4 原始分视图

原始分视图会显示：

1. 每位选手在每位裁判处的总分。
2. 对应的 `+` 分、`-` 分。
3. 如果存在重点扣分，也会显示在单元格中。
4. 该选手的原始平均分。

![](\images\26.png)

### 8.5 高级设置

在折算分视图下，单击“高级设置”可以打开高级设置弹窗。

> 图片占位：高级设置弹窗

可以执行的操作：

1. 修改“比例 (%)”。
2. 勾选或取消“显示重点扣分”。
3. 单击“确定”保存当前显示效果。

注意事项：

1. “显示重点扣分”会按项目保存为本地偏好设置。
2. 同一个项目下再次打开报表时，会自动恢复上次的重点扣分显示状态。

### 8.6 导出详情

单击“导出详情”后，会打开“导出分数详情”窗口。


你可以执行以下操作：

1. 选择要导出的选手。
2. 使用“全选”快速选中全部选手。
3. 勾选导出格式：
   `CSV (日志)`、`SRT (字幕)`。
4. 如果勾选了 `SRT`，再选择 SRT 模式：
   `总分`、`正分 / 负分`、`实时连击`。
5. 单击“下载 ZIP”。

导出结果说明：

1. 软件会下载一个名为 `Details_组名.zip` 的压缩包。
2. 压缩包中会按 `组别 / 选手 / 裁判` 分层保存文件。
3. `CSV (日志)` 文件记录的是时间序列日志。
4. `SRT` 文件适合用于字幕叠加或后期制作。

### 8.7 导出表格 CSV

单击“表格 CSV”会直接下载当前视图对应的 CSV。

规则如下：

1. 如果当前是“折算分”视图，则导出排名、折算分、重点扣分和最终得分。
2. 如果当前是“原始分”视图，则导出选手、各裁判原始分明细和平均分。

### 8.8 视频复盘

视频复盘是首页的独立入口，不会改变当前活动项目。

1. 在首页单击“视频复盘”。
2. 依次选择历史项目、组别和选手。
3. 可按裁判筛选计分事件。
4. 点击已对齐事件后，播放器会切换到事件记录的视频并跳转到对应时间。
5. 播放时会高亮最近的已对齐事件，并按当前视频时间重建每位裁判的累计分数。

事件同步状态说明：

- `已对齐`：可以点击并跳转。
- `未绑定视频`、`播放器未就绪`、`同步过期`或`上下文不一致`：事件仍保留，但不会伪造视频时间。
- 视频不可嵌入、已删除或网络不可用时，事件列表和历史分数仍可查看，并可尝试在系统浏览器中打开视频。

视频复盘中的悬浮计分支持“仅总分”“分项”和“组合显示”，裁判筛选会同步影响显示的计分卡。

## 10. 文件与数据说明

为了方便新用户理解“软件到底把数据保存到哪里”，这里做一个简明说明。

### 10.1 项目数据保存内容

每个项目会保存：

1. 项目配置文件 `config.json`。
2. 各组别目录。
3. 每位选手、每位裁判对应的原始 CSV 记录文件。

典型结构如下：

```text
match_data/
  20260320_120000_My Match/
    config.json
    Group A/
      Player 1_Ref1.csv
      Player 1_Ref2.csv
      Player 2_Ref1.csv
```

### 10.2 设置文件保存内容

全局设置中会保存：

1. 语言。
2. 快捷键。
3. 是否跳过确认。
4. 设备备注。
5. OBS 隔离开关。
6. 各项目的报表偏好设置。

### 10.3 数据保存位置

根据当前代码：

**开发环境**

1. 数据默认保存在项目根目录。
2. 例如当前项目中的 `app_settings.json`、`match_data` 等。

**打包后的正式环境**

1. Windows 通常位于 `%APPDATA%/FT Engine/`
2. macOS 通常位于 `~/Library/Application Support/FT Engine/`

### 10.4 导出文件说明

| 导出类型 | 内容 |
| --- | --- |
| 报表 CSV | 当前表格视图内容 |
| ZIP 详情包 | 选手日志 CSV 与 SRT 字幕文件 |
| 日志 CSV | 时间序列分数变化 |
| SRT 总分 | 显示总分变化 |
| SRT 分项 | 显示 `+ / -` 变化 |
| SRT 实时连击 | 显示短时累计变化 |

## 11. 新用户推荐操作流程

如果你是第一次使用，建议按下面流程操作。

### 11.1 快速上手流程

1. 打开软件。
2. 先到顶部设置中确认语言、快捷键和 OBS 隔离需求。
3. 回到首页，单击“新建比赛”。
4. 先在基础设置中选择比赛模式。

8. 开始比赛，等待设备连接完成。
9. 在计分页中完成打分、切换选手。
10. 如需直播展示，开启悬浮窗。
11. 比赛结束后，从历史记录进入报表页查看和导出数据。

## 12. 使用细节与注意事项

### 12.1 蓝牙与设备相关

1. 如果扫描时报“蓝牙未开启或不可用”，需要先检查电脑蓝牙。
2. 扫描结果会根据 RSSI 排序，信号更强的设备会靠前显示。
3. 设备备注是全局保存的，不是单个项目独有。
4. 永久重命名属于可选操作，不是必须步骤。

### 12.2 比赛流程相关

1. 切换选手时，软件会同步切换比赛上下文并重置当前分数。
2. 自由模式更适合临场快速使用，赛事模式更适合正式项目管理。
3. 计分页中的“自动”会改变快捷键行为，这是实际使用时最需要注意的一点。

### 12.3 悬浮窗相关

1. 悬浮窗打开后默认可穿透，避免挡住原窗口操作。
2. 只有当鼠标移到顶部停靠区或卡片区域时，才能进行拖动和设置。
3. 裁判卡片大小是联动的，调整一张会同步到全部裁判卡。
4. 波形图是独立卡片，可单独拖动和缩放。

### 12.4 数据安全相关

1. 删除项目只删除单个项目数据。
2. 删除本地数据会删除所有项目、所有设置和日志。
3. 比赛结束后建议到历史记录中确认项目已经生成，再进行进一步整理或导出。
